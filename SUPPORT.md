# Support

## Before opening an issue

1. Update to the [latest release](https://github.com/itsreverence/tweet-share/releases/latest).
2. Confirm the userscript is enabled on `x.com` and reload the page.
3. Run **Tweet Discord Share settings…** → **Test webhook**.
4. Try the post from its direct status page rather than a timeline.
5. Check the [README troubleshooting section](README.md#troubleshooting).

## Bug reports

Use the [bug report form](https://github.com/itsreverence/tweet-share/issues/new?template=bug_report.yml). Include the userscript version, browser, userscript manager, reproduction steps, and whether the post is public/protected—without copying private content.

Never include:

- a full Discord webhook URL;
- X/Twitter or Discord cookies, tokens, or session details;
- private/protected post text or media;
- identifying screenshots that have not been redacted.

If a report exposes a security weakness, use the private process in [SECURITY.md](SECURITY.md) instead of a public issue.

## Support boundaries

This repository supports the userscript itself. It cannot provide support for Discord administration, X/Twitter accounts or platform behavior, browser extensions generally, or restoring leaked webhooks. Revoke and replace any webhook you suspect has been exposed.
