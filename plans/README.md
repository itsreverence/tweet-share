## Findings considered and rejected

- **Default `attachMedia: true` without opt-out** — plan 008 keeps one toggle; users can still disable upload. Forced-on with no setting rejected.
- **Faytuks-style custom footer** ("extracted by …") — we use `x.com` host footer + Tweet Share webhook name; low value vs noise.
- **Drop dual-card quote mode** — keep as `quoteLayout: "card"` for both-media attribution.
- **Server-side mirror bot** — out of scope; userscript-only architecture stands.
- **“Video in embed fields is broken”** — by design; attachments or separate message required.
